# AI Creator — 你的 AI 小说创作工作台

[English](./README.en.md)

> 不是又一个"AI 聊天 + 编辑器"。
> AI Creator 内置一位真正懂你故事的 AI 编辑，它能管理你的角色和设定、续写章节、导入参考资料、在创作知识库中语义搜索——像一个经验丰富的写作搭档，而不是一个只会接话的聊天机器人。

---

## 核心能力：你的 AI 写作搭档

AI Creator 的核心是一个拥有 **18 个专用工具** 的 AI Agent。它不只是生成文字——它理解你的世界观、记住你的偏好、主动查阅已有设定，确保每一次创作都前后一致。

### 角色与世界观管理

通过对话直接操作你的创作数据库：

- **创建 / 编辑 / 删除角色**——支持外貌、性格、背景、目标、人物关系等多维度描写，按重要性（核心 / 主要 / 次要）分级管理。
- **创建 / 编辑 / 删除世界观条目**——地理、历史、魔法体系、政治制度、科技、文化、组织势力……分类清晰，检索方便。
- **创建 / 编辑 / 删除草稿**——灵感片段、大纲备忘、参考笔记，还能关联到具体角色或设定条目。

> 你说"帮我创建一个住在北城的铁匠，性格暴躁但心地善良"，AI 就会调用工具写入数据库，下次续写时自动引用。

### 智能续写

不是简单的"接着往下编"：

- **上下文感知**——续写前自动获取当前章节全文 + 前两章内容，理解剧情走向。
- **风格一致**——分析已有章节的叙事口吻、用词习惯和节奏感，续写时保持风格统一。
- **可控方向**——你可以指定续写字数、剧情走向和特殊指令。
- **无缝写入**——生成的文字直接追加到编辑器中，所见即所得。

### 语义搜索

在你的整个创作知识库中做智能检索：

- 跨角色、世界观、草稿、章节全库搜索。
- 支持向量语义匹配（需配置 Embedding 服务），也可退化为正则匹配。
- "那个和主角在第三章结仇的将军叫什么？"——AI 会搜索后直接告诉你。

### 双层记忆系统

AI 不会每次对话都"失忆"：

- **世界观级记忆**——跨所有小说共享的偏好和规则（例如"这个世界里没有枪械"）。
- **小说级记忆**——单部作品的特定指令（例如"主角的叙述视角始终用第一人称"）。
- 记忆自动注入每次对话上下文，也可以在 UI 中手动编辑。

### 章节摘要自动生成

- 章节内容变更后自动标记为"待生成"，后台由 AI 重新生成摘要。
- 摘要生成时会参考前序章节（最近 5 万字全文 + 历史摘要），保持剧情连贯。
- 生成的摘要保留人名、目标、冲突、转折等关键细节，方便回顾长篇剧情线。

### 文件导入与知识提取

把已有的参考资料变成结构化创作数据：

- 支持 `.txt`、`.md`、`.docx`、`.pdf` 格式。
- 大文件自动分块，AI 逐块提取人物、地点、势力等信息。
- 提取过程中自动去重（语义搜索已有条目避免重复创建）。
- 全程流式进度反馈，断点可续传。

### 对话上下文压缩

长时间创作不怕对话过长：

- 上下文接近模型窗口上限时自动压缩历史对话。
- 压缩保留最近 2 轮完整对话，之前的部分由 AI 生成高质量摘要。
- 摘要包含：工作目标、关键约束、已完成事项、待处理问题、涉及的角色和设定——不会丢失重要信息。

### 用量透明

- 每轮对话实时显示 Token 用量和费用。
- 区分输入 / 输出 / 缓存读取 / 缓存写入，清晰掌控成本。

## 沉浸式写作体验

写小说需要氛围感：

- **动态背景**——全屏 WebGL 着色器渲染的雨夜玻璃和星空动效。
- **毛玻璃界面**——半透明面板搭配动态背景，营造沉浸创作氛围。
- **11 种精选字体**——霞鹜文楷、龙藏体、马善政楷书、站酷小薇等，一键切换。
- **浮动控制栏**——章节导航、AI 聊天、主题切换，不打断你的写作心流。
- **智能字数统计**——中文自动显示「字」数，英文显示词数。

## 多端同步，随时创作

- **Web 端**——左侧编辑器 + 右侧 AI 对话，双栏并行的完整工作台。
- **桌面端**——Electron 封装，独立窗口沉浸写作。
- **移动端**——Expo 原生应用，支持章节编辑、AI 对话、设定管理。外出时记录灵感，回到电脑前无缝继续。

## 灵活的模型选择

支持 OpenAI、Anthropic、Google 等多家供应商。可配置模型列表、默认模型和推理强度级别，不同用户组可有不同的模型权限。

## 快速开始

### 环境要求

- Node.js >= 20、pnpm >= 9、MongoDB

### 安装与运行

```bash
pnpm install
cp apps/server/.env.example apps/server/.env
# 编辑 .env，填写 MONGODB_URI、JWT_SECRET 和至少一个 LLM API Key

pnpm build        # 首次运行前构建
pnpm dev:all      # 启动后端 (3001) + Web (5173)
```

移动端和桌面端：

```bash
# 桌面端
pnpm dev:web && pnpm --filter @ai-creator/desktop dev

# 移动端
pnpm --filter @ai-creator/mobile dev
```

### 环境变量

在 `apps/server/.env` 中配置：

| 变量 | 说明 |
|------|------|
| `MONGODB_URI` | MongoDB 连接字符串（必填） |
| `JWT_SECRET` | JWT 签名密钥（必填） |
| `LLM_API_KEY` | 通用 LLM API Key，或使用 `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `GOOGLE_API_KEY`（至少填一个） |
| `AVAILABLE_MODELS` | 可选模型列表，格式 `provider:modelId`，逗号分隔 |
| `DEFAULT_MODEL` | 默认模型，格式 `provider:modelId` |
| `EMBEDDING_*` | Embedding 服务配置（可选，启用语义搜索） |
